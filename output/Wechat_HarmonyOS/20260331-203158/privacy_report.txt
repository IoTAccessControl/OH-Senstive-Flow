1 我们如何收集和使用您的个人信息

在通讯录功能中，我们仅在本地读取并处理您的联系人姓名和联系人头像标识，用于展示通讯录列表及支持您点击联系人发起聊天；相关数据仅在本地处理。

在首页展示时，我们会通过系统接口收集您的设备市场名称、操作系统全名、SDK API 版本、设备类型、系统运行时间、屏幕信息（宽度、刷新率、密度）、支持的传感器 ID 列表、默认网络激活状态、网络承载类型及 IP 地址等信息，用于本地显示设备硬件参数、系统信息及网络状态详情。相关数据仅在本地处理，仅存储在组件内存状态变量中，未进行本地持久化存储或加密，也不会上传至应用服务端。

在聊天列表功能中，我们会收集并使用您的用户昵称和聊天列表数据，用于渲染聊天列表界面并在您点击时跳转至对应聊天页面。相关数据仅在本地处理。

当您点击头像更换个人图片时，我们会从系统相册获取您选择的用户头像图片URI，用于更新您在“我的”页面显示的头像；若选择失败，我们将展示相册选择错误信息以提示您。相关数据仅在本地处理。

在搜索页面，我们会收集您手动输入的“用户搜索关键词”，该数据仅在内存中临时存储并用于本地界面展示以响应用户的搜索请求，相关数据仅在本地处理。

2 设备权限调用

在“页面展示时获取网络连接状态及详细属性”场景中，我们会调用获取网络信息权限（预授权），用于检查默认网络是否存在，获取网络类型、IP 地址、子网掩码及网络能力集。若您拒绝授权，无法显示当前网络类型、IP 地址、子网掩码等网络详细信息。

在“用户点击头像更换个人图片”场景中，我们会调用读取媒体内容权限（动态授权），用于访问系统相册以获取用户选择的图片。若您拒绝授权，无法打开相册选择器，导致用户无法更换头像。

当前已在应用源码/配置扫描或 SDK API 权限映射中识别到以下权限：相机权限（动态授权）、网络访问权限（预授权）、麦克风权限（动态授权）、振动权限（预授权）、写入媒体内容权限（动态授权）；但尚未定位到可回溯的功能点数据流，因此本章节暂不生成具体权限声明。
