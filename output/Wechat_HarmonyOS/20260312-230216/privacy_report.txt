1 我们如何收集和使用您的个人信息

在“应用退出时”场景中，我们会从系统事件收集应用生命周期状态事件、日志标签 (testTag)和格式化日志字符串，用于调试与系统监控。

在“应用启动并创建主窗口时，系统记录页面加载状态及错误信息以辅助调试”场景中，我们会从应用内部运行状态、页面加载回调数据收集应用生命周期状态事件、页面加载返回数据和页面加载错误对象，用于记录应用启动及页面加载过程中的状态与异常信息，用于开发调试。

在“应用退出时”场景中，我们会从系统事件收集应用生命周期状态事件，用于调试与运行状态监控。

在“应用切到前台时”场景中，我们会从系统事件收集应用生命周期状态事件 (Foreground)，用于记录应用进入前台的状态信息用于调试或监控。

在“应用切到后台时”场景中，我们会从系统事件收集应用生命周期状态事件和静态日志字符串，用于记录应用进入后台的状态信息用于调试或监控。

在“聊天页面加载时获取聊天对象名称，以及用户发送消息时处理文本或语音转文字内容”场景中，我们会从应用内部路由参数、用户输入收集聊天对象名称、聊天消息内容（文本/语音转文字）和键盘高度状态，用于展示聊天界面、显示聊天对象名称、记录并展示用户发送的消息内容、调整 UI 布局以适应软键盘。

在“用户在聊天页面点击语音输入图标，将语音转换为文本消息并发送”场景中，我们会从用户设备麦克风、用户输入收集用户聊天消息内容 (text/voice)、联系人/聊天对象名称和麦克风音频数据 (潜在)，用于实现语音转文字功能以生成聊天消息。

在“用户在工具栏点击搜索图标或添加图标时，应用会捕获该交互事件并执行页面跳转逻辑”场景中，我们会从用户界面交互收集用户交互事件（点击操作）和页面导航目标 URL，用于实现应用内不同功能模块间的页面导航与切换。

在“用户在通讯录页面查看联系人列表并点击特定联系人以进入聊天界面时”场景中，我们会从本地内存数据收集联系人姓名 (contact.name)、联系人头像 (contact.head)和通讯录分组标题 (item.title)，用于展示通讯录列表内容并根据用户选择跳转至对应聊天会话。

在“用户在通讯录页面查看联系人列表，点击字母索引快速跳转，或点击具体联系人进入聊天界面”场景中，我们会从本地内存数据源 (this.dataList)、用户交互事件收集联系人姓名、联系人头像、通讯录分组标题、通讯录列表索引位置和字母索引标签，用于展示联系人列表供用户浏览，支持通过字母索引快速定位联系人，并允许用户选择联系人发起聊天。

在“在通讯录页面展示联系人列表并支持点击进入聊天界面”场景中，我们会从本地通讯录数据收集联系人姓名 (contact.name)、联系人头像 (contact.head)和通讯录列表数据 (this.dataList)，用于向用户展示其通讯录中的联系人信息，并支持选择特定联系人发起会话。

在“用户在首页查看聊天列表并点击特定会话进入聊天详情页时”场景中，我们会从本地内存数据收集联系人昵称 (nickname)、聊天列表数据 (chatList)、头像资源标识 (headSrc)、聊天内容预览 (content)和时间戳 (time)，用于展示聊天会话列表并支持用户点击进入具体聊天界面。

在“用户在“我的”页面点击头像以更换个人头像图片”场景中，我们会从系统相册/媒体库收集用户头像图片 URI、用户头像图片文件内容、应用包信息 (BundleInfo)和访问令牌 ID (AccessTokenId)，用于允许用户自定义和展示个人头像，完善个人资料信息。

在“用户在搜索页面输入关键词进行搜索，或点击取消/返回按钮退出当前页面时”场景中，我们会从用户输入、系统状态收集用户输入的搜索文本、键盘高度数值、页面路由栈深度和物理按键事件类型，用于实现搜索功能交互、控制软键盘显示与隐藏、处理页面导航逻辑及提供操作反馈。

2 设备权限调用

在“用户在聊天界面使用语音输入功能时”场景中，我们会调用ohos.permission.MICROPHONE权限，用于获取麦克风权限以录制用户语音并进行转文字处理。若您拒绝授权，无法使用语音输入功能，需切换为键盘输入。

在“Checks whether the default data network is activated.”场景中，我们会调用ohos.permission.GET_NETWORK_INFO权限，用于使用相关系统能力（由 SDK API 权限映射确定）。若您拒绝授权，拒绝授权可能导致对应功能无法正常使用。

在“用户在“我的”页面点击头像以选择照片作为新头像时”场景中，我们会调用ohos.permission.READ_MEDIA权限，用于读取用户设备上的媒体文件（如照片），以便用户选择并设置为个人头像。若您拒绝授权，无法打开系统相册选择图片，导致无法更换头像。此外，在“用户在“我的”页面点击头像以选择照片作为新头像时”场景中，我们会调用ohos.permission.WRITE_MEDIA权限，用于写入或修改媒体文件相关数据，配合读取权限完成图片选择及后续可能的缓存操作。若您拒绝授权，无法完成图片选择流程，导致无法更换头像。

当前已在应用源码/配置扫描或 SDK API 权限映射中识别到以下权限：ohos.permission.CAMERA、ohos.permission.INTERNET、ohos.permission.VIBRATE；但尚未定位到可回溯的功能点数据流，因此本章节暂不生成具体权限声明。
