1 我们如何收集和使用您的个人信息

在“应用退出时”场景中，我们会从系统日志缓冲区收集应用生命周期状态事件、日志标签 (testTag)和固定日志消息字符串，用于记录应用销毁时的系统状态信息。

在“应用启动并创建主窗口时，记录系统状态及页面加载结果日志”场景中，我们会从应用内部运行状态、页面加载回调数据收集应用生命周期状态事件、页面加载错误详情和页面加载返回数据，用于开发调试及监控应用启动与页面加载过程中的异常状态。

在“应用退出时”场景中，我们会从系统事件收集应用生命周期状态事件、静态日志标签 (testTag)和固定描述字符串 (Ability onWindowStageDestroy)，用于记录应用窗口销毁时的系统状态以辅助资源释放与调试。

在“应用切到前台时”场景中，我们会从系统事件收集应用生命周期状态事件和日志文本字符串，用于记录应用进入前台的生命周期事件，用于调试或运行状态监控。

在“应用切到后台时”场景中，我们会从系统事件收集应用生命周期状态事件和日志文本字符串，用于记录应用进入后台的状态信息用于调试或监控。

在“聊天页面加载及消息发送时”场景中，我们会从用户输入、系统路由参数、系统事件收集聊天对象名称、键盘高度状态、聊天消息内容、应用访问令牌和应用包信息，用于初始化聊天界面标题、适配软键盘布局、构建并暂存待发送的聊天消息、校验功能所需权限状态。

在“用户在聊天页面点击语音输入图标，准备进行语音消息录制时”场景中，我们会从设备麦克风收集麦克风音频数据 (潜在)和用户授权状态，用于支持用户在聊天界面通过语音输入发送消息。

在“用户在工具栏界面点击搜索图标或添加图标时，触发页面跳转或显示功能开发提示”场景中，我们会从用户交互事件（点击操作）、本地 UI 组件状态收集用户点击事件、页面导航目标 URL、UI 组件状态 (title)和静态提示文本，用于响应用户交互操作，实现应用内页面导航及功能状态提示。

在“用户在通讯录页面查看联系人列表并点击特定联系人以进入聊天界面时”场景中，我们会从本地内存数据收集联系人姓名、联系人头像和通讯录分组标题，用于展示通讯录列表内容，并根据用户选择跳转到对应的聊天会话页面。

在“用户在通讯录页面点击字母索引导航时，应用根据点击位置计算并滚动列表至对应分组”场景中，我们会从用户交互事件（点击索引器）、本地通讯录数据列表收集通讯录列表索引位置 (List Index)和字母索引标签 (Letter Index)，用于响应用户点击操作，快速定位并展示通讯录中对应的联系人分组。

在“用户在通讯录页面查看联系人列表，或点击特定联系人进入聊天界面时”场景中，我们会从应用内部内存数据（this.dataList）收集联系人姓名 (contact.name)、联系人头像 (contact.head)和分组标题 (item.title)，用于展示通讯录列表供用户浏览，并在用户选择联系人时启动对应的聊天会话。

在“用户进入硬件信息页面查看设备详情时，应用自动采集并展示设备基础参数、屏幕规格、传感器列表及当前网络连接状态”场景中，我们会从系统设备信息接口、系统显示服务接口、系统传感器服务接口、系统网络管理接口收集设备市场名称、操作系统全称、SDK API 版本、设备类型、系统运行时间、屏幕分辨率（宽高）、屏幕刷新率、屏幕像素密度、支持的传感器 ID 列表、网络承载类型、IP 地址和子网掩码，用于向用户展示当前设备的硬件配置、系统版本、屏幕参数及网络连接状态等详细信息。

在“在微信首页展示聊天列表，并响应用户点击以进入具体聊天会话”场景中，我们会从本地内存存储收集联系人昵称 (nickname)、聊天内容预览 (content)和最后消息时间 (time)，用于向用户展示最近的聊天会话列表，支持用户选择并跳转至具体聊天界面。

在“用户在个人资料页点击头像以更换个人头像图片”场景中，我们会从用户设备相册（系统媒体库）收集用户头像图片文件及 URI、应用包信息 (BundleInfo)和访问令牌 ID (AccessTokenId)，用于允许用户自定义并展示个人资料页的头像图片。

在“搜索页面展示时，应用监听系统软键盘高度变化以适配界面布局”场景中，我们会从系统事件收集键盘高度变化事件和窗口对象引用，用于获取软键盘高度信息，用于动态调整搜索页面的显示区域和布局，确保用户输入体验。

在“用户在搜索页面输入关键词进行搜索时”场景中，我们会从用户手动输入收集用户输入的搜索文本 (searchText)，用于响应用户搜索操作，提供即时反馈与页面导航控制。

2 设备权限调用

在“用户在聊天页面点击语音输入图标，准备进行语音消息录制时”场景中，我们会调用ohos.permission.MICROPHONE权限，用于获取麦克风权限以录制用户的语音内容用于发送消息。若您拒绝授权，若拒绝授权，用户将无法使用语音输入功能发送语音消息。

在“用户进入硬件信息页面查看设备详情时”场景中，我们会调用ohos.permission.GET_NETWORK_INFO权限，用于获取当前默认网络连接状态、网络类型（WiFi/蜂窝）、IP 地址及子网掩码等网络配置信息。若您拒绝授权，若拒绝授权，应用无法访问网络接口，导致页面中网络类型、IP 地址等网络相关信息无法显示。

在“用户在个人资料页点击头像以选择新图片时”场景中，我们会调用ohos.permission.READ_MEDIA权限，用于读取用户设备相册中的图片文件，以便将其设置为新的个人头像。若您拒绝授权，无法打开系统相册选择器，导致用户无法更换头像。此外，在“用户在个人资料页点击头像以选择新图片时”场景中，我们会调用ohos.permission.WRITE_MEDIA权限，用于获取写入权限以支持系统相册选择器的交互流程及后续可能的文件操作。若您拒绝授权，无法完成相册选择流程，导致用户无法更换头像。

当前已在应用源码/配置扫描或 SDK API 权限映射中识别到以下权限：ohos.permission.CAMERA、ohos.permission.INTERNET、ohos.permission.VIBRATE；但尚未定位到可回溯的功能点数据流，因此本章节暂不生成具体权限声明。
