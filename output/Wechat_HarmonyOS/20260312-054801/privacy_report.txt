1 我们如何收集和使用您的个人信息

在“应用退出时”场景中，我们会从系统日志收集应用生命周期状态事件、日志标签 (testTag)和格式化日志字符串，用于记录应用销毁时的状态信息用于调试或监控。

在“应用启动并创建主窗口时”场景中，我们会从系统日志收集页面加载状态与错误详情，用于记录应用启动及页面加载过程中的状态与异常信息。

在“应用退出时”场景中，我们会从系统事件收集应用生命周期状态事件、固定日志标签 (testTag)和静态描述字符串 (Ability onWindowStageDestroy)，用于记录应用窗口销毁时的系统状态信息用于调试或监控。

在“应用切到前台时”场景中，我们会从系统事件收集应用生命周期状态事件 (Ability onForeground)，用于记录应用进入前台的生命周期事件。

在“应用切到后台时”场景中，我们会收集应用生命周期状态事件和日志文本字符串，用于记录应用运行时的关键生命周期节点，用于调试与问题排查。

在“进入聊天界面时，应用从路由参数中获取当前聊天对象的名称以显示在工具栏，并监听系统键盘高度变化以调整页面布局”场景中，我们会从路由参数、系统事件收集聊天对象名称和键盘高度状态，用于展示聊天对象信息；适配软键盘弹出时的界面布局。此外，在“用户在聊天界面构建消息（如语音转文字）或准备发送时，应用检查并动态申请必要的系统权限（如麦克风、相机等），并将生成的消息内容暂存于本地列表”场景中，我们会从用户输入、系统权限服务、应用包信息收集聊天消息内容、应用访问令牌 (AccessTokenId)、应用包信息 (BundleInfo)和用户授权状态，用于构建待发送的聊天消息；验证应用身份以执行敏感操作；获取用户授权以使用设备功能。

在“用户在聊天界面使用语音输入功能时，应用会收集用户的语音数据并可能转换为文本消息”场景中，我们会从用户设备麦克风、本地内存收集语音输入内容 (voiceToText)、聊天消息列表 (chatList/dataList)和用户昵称 (name)，用于实现即时通讯中的语音转文字及消息发送功能。

在“用户在工具栏点击搜索图标或添加图标时，应用会捕获该交互事件并执行页面跳转逻辑”场景中，我们会从用户界面操作收集用户交互事件（点击操作）和页面导航目标 URL，用于实现应用内不同功能模块间的页面导航与切换，响应用户的操作意图。

在“用户在通讯录页面查看联系人列表并点击特定联系人以进入聊天界面时”场景中，我们会从应用内部数据源收集联系人姓名、联系人头像和通讯录列表数据，用于展示通讯录列表内容并支持用户选择联系人发起聊天会话。

在“在通讯录列表页面展示联系人信息，支持按字母索引快速跳转及点击联系人进入聊天”场景中，我们会从本地通讯录数据收集通讯录列表数据 (this.dataList)、联系人姓名 (contact.name)、联系人头像 (contact.head)和通讯录列表索引位置 (List Index)，用于展示通讯录列表、支持字母索引跳转及进入单聊界面。

在“用户在通讯录页面点击联系人以进入聊天界面时”场景中，我们会从本地内存存储 (ArkUI 组件状态 dataList)收集联系人姓名 (contact.name)、联系人头像 (contact.head)和通讯录列表数据 (this.dataList)，用于展示联系人列表并支持用户选择特定联系人发起聊天会话。

在“在硬件信息页面加载时，自动收集并展示设备的硬件配置、系统版本、屏幕参数及网络连接状态”场景中，我们会从设备市场名称、操作系统全称、SDK API 版本、设备类型、系统运行时间、屏幕高度、屏幕宽度、屏幕刷新率、屏幕像素密度、支持的传感器 ID 列表、默认网络连接状态、网络承载类型、IP 地址、子网掩码收集设备市场名称 (marketName)、操作系统全称 (osFullName)、SDK API 版本 (sdkApiVersion)、设备类型 (deviceType)、系统运行时间 (uptime)、屏幕分辨率与参数 (displayHeight, displayWidth, displayRefreshRate, displayDensityDPI)、支持的传感器 ID 列表 (supportSensorIds)、网络承载类型 (netBearType)、IP 地址 (IPAddress)和子网掩码 (subnetMask)，用于向用户展示当前设备的硬件规格、系统环境及网络连接详情，以便用户了解设备状态。

在“在应用首页展示聊天会话列表，并响应用户点击操作以跳转至具体聊天详情页面”场景中，我们会从本地内存数据源 (this.chatList)收集联系人昵称、聊天内容预览和消息时间，用于向用户展示最近的聊天会话信息，并支持用户进入具体的聊天对话场景。

在“用户点击个人主页头像以更新或查看个人信息时”场景中，我们会从系统相册/媒体库收集用户头像图片 URI和用户头像图片路径 (imgPath)，用于允许用户从系统相册选择并设置个人头像，并在个人中心页面展示。

在“用户在搜索页面输入关键词进行搜索，或点击取消/返回按钮退出当前页面时”场景中，我们会从用户输入、系统事件收集用户输入的搜索文本、键盘高度状态、页面路由栈深度和用户按键事件，用于实现搜索功能交互、控制页面导航（返回或退出应用）以及响应物理按键操作。

2 设备权限调用

在“当用户点击或触发语音输入功能时，需要获取麦克风权限以录制音频”场景中，我们会调用ohos.permission.MICROPHONE权限，用于采集用户语音数据以进行语音识别或发送语音消息。若您拒绝授权，如果拒绝授权，用户将无法使用语音输入功能，只能使用纯文本输入。

在“当用户进入硬件信息页面时，应用需要获取当前的网络连接属性（如 IP 地址、网络类型）”场景中，我们会调用ohos.permission.GET_NETWORK_INFO权限，用于调用 connection.getDefaultNetSync() 和 getConnectionPropertiesSync() 等 API 以获取详细的网络链路信息。若您拒绝授权，如果拒绝此权限，应用可能无法正确获取并显示 IP 地址、子网掩码及具体的网络承载类型，导致相关硬件信息显示为空白或错误。

在“用户点击头像触发相册选择器时”场景中，我们会调用ohos.permission.READ_MEDIA权限，用于申请读取手机上的媒体文件（如照片）权限，以便用户选择新头像。若您拒绝授权，如果拒绝此权限，用户将无法打开相册选择器，导致无法更换头像。此外，在“用户点击头像触发相册选择器时”场景中，我们会调用ohos.permission.WRITE_MEDIA权限，用于申请向媒体库写入数据的权限，可能用于保存选中的图片或临时缓存。若您拒绝授权，如果拒绝此权限，应用可能无法完成图片的加载、预览或本地缓存操作。

当前已在应用源码/配置扫描或 SDK API 权限映射中识别到以下权限：ohos.permission.CAMERA、ohos.permission.INTERNET、ohos.permission.VIBRATE；但尚未定位到可回溯的功能点数据流，因此本章节暂不生成具体权限声明。
