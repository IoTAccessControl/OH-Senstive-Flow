1 我们如何收集和使用您的个人信息

在“用户在搜索栏界面查看默认搜索关键词或点击搜索框时，应用读取本地存储的历史记录并展示在输入框中，或在点击时跳转至搜索页面”场景中，我们会从本地持久化存储收集searchDefault (搜索默认关键词/历史)和isShowPage (页面显示状态)，用于恢复用户之前的搜索偏好设置、控制搜索框交互状态及实现页面跳转。

在“用户在首页点击视频条目时，应用将视频详情数据传递至详情页进行展示”场景中，我们会从应用内部组件状态收集视频元数据 (BaseVideo)、视频封面 URL、视频标题和视频 ID，用于响应用户点击操作，实现从列表页到详情页的跳转与内容展示。

在“用户在推荐列表点击视频条目时，应用将视频元数据传递至详情页进行展示”场景中，我们会从应用内部组件状态收集视频元数据 (vInfo)、视频封面 URL (pic)、视频标题、视频作者和视频播放数/弹幕数，用于渲染视频推荐卡片及跳转至视频详情页展示内容。

在“应用启动及闪屏页展示时，初始化设备标识并检查媒体相关权限状态”场景中，我们会从本地存储、系统日志收集appUuid (设备唯一标识)和appLogDebug (调试配置)，用于标识当前设备或安装实例，以便区分首次启动与后续启动，并控制日志输出级别。

在“用户在搜索栏输入关键词或点击搜索按钮时，应用收集并传递搜索内容以进行页面跳转和后续处理”场景中，我们会从用户输入、组件内部状态收集用户输入的搜索关键词 (keyword)和默认占位符文本 (placeholder)，用于响应用户搜索请求，将搜索关键词传递至搜索结果页面以展示匹配内容。

在“应用启动初始化时”场景中，我们会从系统日志接口、应用上下文对象收集应用生命周期状态、全局上下文对象 (appContext)和资源管理器实例 (resourceManager)，用于记录应用启动状态并初始化全局图像处理工具链。

在“应用退出时”场景中，我们会从应用内部状态收集Ability 生命周期状态字符串，用于记录应用销毁时的调试信息。

在“应用启动初始化时，记录系统日志、配置本地存储路径及网络状态监听”场景中，我们会从应用上下文信息、系统生命周期事件、网络连接状态回调收集应用文件目录路径 (filesDir)、应用包信息 (AppBundleInfo)、网络状态标识 (netIsAvailable)、网络连接 ID (netId)、页面加载错误详情 (Error Object)、页面加载成功数据 (Load Content Data)和生命周期事件日志字符串，用于初始化应用运行环境，记录启动过程日志，监控网络连接状态以支持后续业务功能。

在“应用退出时”场景中，我们会从应用内部状态收集应用生命周期状态事件和固定日志标签字符串，用于记录应用窗口销毁时的状态信息用于调试或监控。

在“应用切到前台时”场景中，我们会从系统事件收集应用生命周期状态事件 (Foreground)，用于记录应用进入前台的状态信息用于调试或监控。

在“应用切到后台时”场景中，我们会从应用内部状态收集应用生命周期状态事件和日志文本字符串，用于记录应用进入后台的状态信息用于调试或监控。

在“用户在查看日志详情页面时，应用加载并展示指定的本地日志文件内容”场景中，我们会从本地文件系统收集本地日志文件内容和日志文件路径，用于向用户展示日志文件的详细文本内容以便排查问题或查看运行记录。

在“视频播放页加载与播放控制”场景中，我们会从设备显示信息、用户输入的视频标识、网络请求响应收集屏幕分辨率 (宽高)、屏幕刘海/挖孔区域信息、全屏显示比例计算值、视频标识符 (bvid)、分集标识符 (cid)、视频播放地址 (url)、播放器状态事件、播放进度时间戳和弹幕数据模型，用于适配不同屏幕尺寸以优化视频显示效果，获取并播放指定的视频内容，同步播放进度与弹幕。

在“用户在视频播放页面点击弹幕开关按钮时，应用读取当前 UI 状态变量以控制弹幕显示或隐藏”场景中，我们会从用户交互事件、内存中的 UI 状态变量收集UI 状态变量 (mIsShowCtl, isPortrait)和用户交互事件 (点击操作)，用于响应用户对弹幕开关的操作，动态控制视频播放器界面上弹幕的显示与隐藏。

在“用户进入搜索页面时，应用加载热门搜索榜单并展示默认搜索词；用户点击热搜关键词或完整榜单入口时，将相关数据传递至结果页进行后续处理”场景中，我们会从路由参数 (Router Parameters)、云端 API 响应 (Cloud API Response)收集默认搜索关键词、热搜榜单数据和用户选中的搜索关键词，用于展示热门搜索内容，响应用户搜索意图并提供跳转服务。

在“用户在搜索结果页加载时，应用获取搜索关键词并发起网络请求以展示搜索结果”场景中，我们会从用户输入、页面路由参数收集用户搜索关键词 (keyword)和页面路由参数 (router params)，用于执行用户搜索请求并返回相关结果列表。

在“用户进入搜索排行榜页面时，应用接收并展示热搜榜单数据；用户点击返回按钮退出当前页面；用户点击榜单项跳转至搜索结果页”场景中，我们会从上一页传递的路由参数收集页面路由参数 (TopSearchParams)、排行榜数据 (rankings)、搜索关键词 (keyword)和列表项位置 (position)，用于展示热搜榜单内容，响应用户交互以返回上一页或进入搜索结果详情。

2 设备权限调用

在“应用启动及闪屏页展示时，准备获取用户媒体文件的位置信息”场景中，我们会调用ohos.permission.MEDIA_LOCATION权限，用于访问用户媒体文件关联的地理位置信息。若您拒绝授权，若拒绝授权，应用可能无法获取媒体文件的地理位置信息，但根据代码逻辑，应用仍会尝试跳转至主页面，具体功能受限情况取决于后续业务逻辑。此外，在“应用启动及闪屏页展示时，准备读取用户媒体文件内容”场景中，我们会调用ohos.permission.READ_MEDIA权限，用于读取用户设备上的媒体文件（如图片、视频等）。若您拒绝授权，若拒绝授权，应用将无法读取用户媒体文件，可能导致依赖媒体内容的功能（如推荐内容拉取）无法正常工作。

在“应用启动初始化时”场景中，我们会调用ohos.permission.INTERNET权限，用于创建网络连接实例并注册网络状态监听。若您拒绝授权，无法建立网络连接，无法监听网络可用/不可用状态变化。

在“用户在查看日志详情页面时，应用尝试读取可能位于媒体目录下的日志文件”场景中，我们会调用ohos.permission.READ_MEDIA权限，用于访问并读取本地存储中的日志文件内容。若您拒绝授权，若拒绝该权限，应用可能无法读取位于受保护媒体目录下的日志文件，导致日志详情页面无法显示内容或显示为空。

在“视频播放过程中请求资源”场景中，我们会调用ohos.permission.INTERNET权限，用于向服务器发送视频标识 (bvid/cid) 以获取分集列表和真实播放地址，以及获取弹幕数据。若您拒绝授权，无法加载视频列表、无法获取视频播放源、无法显示弹幕，导致视频无法播放。

在“用户进入搜索页面时，应用从云端获取热搜榜单数据”场景中，我们会调用ohos.permission.INTERNET权限，用于发起网络请求以获取热搜榜单列表。若您拒绝授权，无法加载和展示热门搜索榜单数据，仅能显示本地缓存或空状态。
