1 我们如何收集和使用您的个人信息

在“用户在搜索栏界面进行交互时，应用读取本地存储的默认搜索关键词和页面显示状态，用于初始化输入框占位符及控制焦点行为；当用户点击搜索栏时，将默认关键词传递至搜索页面”场景中，我们会从本地存储 (Preferences)收集searchDefault和isShowPage，用于优化搜索体验，提供默认搜索词提示并控制输入框交互状态。

在“用户在首页点击视频推荐项时，应用读取并展示视频封面、标题等元数据，并在用户交互后将完整视频信息传递至播放页面”场景中，我们会从应用内部组件状态 (this.item)收集视频信息对象 (BaseVideo)、视频封面图片 URL、视频标题、视频播放数/弹幕数统计和视频 ID，用于展示视频列表内容及响应用户点击跳转至详情页。

在“在关于页面展示与交互过程中，当用户触发推荐视频项的点击操作时，系统会收集并传递当前选中视频的元数据（包括封面、标题、ID）至目标播放或详情页面”场景中，我们会从应用内部状态收集视频元数据 (vInfo)、视频封面 URL (pic)、视频标题/描述和视频 ID，用于实现从推荐列表到视频详情页目的页面跳转与内容加载。

在“用户在搜索栏输入关键词或点击搜索按钮时，应用收集并处理搜索请求以展示搜索结果”场景中，我们会从用户手动输入、默认占位符文本收集用户搜索关键词 (keyword)和默认占位符文本 (placeholder)，用于响应用户搜索请求，展示匹配的搜索结果内容。

在“应用启动初始化时”场景中，我们会从应用内部上下文对象、系统资源管理器收集Ability 上下文对象 (Context)、资源管理器实例 (ResourceManager)和图片处理引擎配置信息，用于初始化应用核心环境、加载资源及配置图片处理引擎，确保应用正常启动与运行。

在“应用退出时”场景中，我们会从系统日志缓冲区收集应用生命周期状态事件和日志文本字符串，用于记录应用正常退出的生命周期状态，用于调试与资源清理确认。

在“应用启动初始化时”场景中，我们会从系统日志接口、全局内存变量、应用级存储 (AppStorage)、本地文件系统路径收集应用文件目录路径、窗口阶段对象引用、网络连接状态标志、网络标识符 (netId)、应用包名、应用版本号、应用目标版本、应用厂商信息、页面加载错误信息和页面加载回调数据，用于初始化应用运行环境、配置日志存储路径、监控网络连接状态以及获取应用自身版本信息以进行兼容性检查。

在“应用退出时”场景中，我们会从应用内部生命周期状态收集生命周期事件状态字符串、日志标签和日志域标识，用于记录应用窗口销毁时的系统状态以辅助调试。

在“应用切到前台时”场景中，我们会从系统事件收集应用生命周期状态事件和日志文本字符串，用于记录应用进入前台的状态信息用于调试或监控。

在“应用切到后台时”场景中，我们会从系统事件触发收集应用生命周期状态事件和日志文本字符串，用于开发调试，追踪应用进入后台时的运行状态。

在“用户在反馈页面查看日志文件列表并选择特定日志进行详情跳转或上传”场景中，我们会从本地文件系统收集本地日志文件路径 (logFilePath)、日志文件名 (logName)、应用唯一标识 (appUuid)和用户反馈文本内容，用于收集设备运行日志以辅助问题定位和软件调试。

在“用户在反馈页面查看日志列表、切换调试模式或提交问题时，应用收集本地日志文件元数据及内容并上传至云端”场景中，我们会从应用私有沙箱目录 (filesDir/log)、本地持久化存储 (@StorageLink)收集应用唯一标识 (appUuid)、调试模式状态 (appLogDebug)、应用日志文件列表、日志文件大小、日志文件路径和日志文件内容，用于问题定位、故障排查及接收用户反馈，将设备运行日志发送至开发者服务器进行分析。

在“用户查看日志详情页面时，应用加载并展示指定的本地日志文件内容”场景中，我们会从路由参数、本地文件系统收集日志文件路径和本地日志文件内容，用于在界面上渲染并展示日志文本内容供用户查阅。

在“用户查看日志详情页面时，应用加载并展示本地存储的日志文件内容”场景中，我们会从本地文件系统收集本地日志文件内容和路由传递参数 (logFilePath)，用于在界面上渲染并展示日志文本供用户阅读。

在“视频播放页面加载时，为适配不同屏幕（特别是处理刘海屏/挖孔屏）以优化视频显示效果”场景中，我们会从设备显示屏信息收集屏幕分辨率 (width/height)、屏幕刘海/挖孔区域信息 (CutoutInfo)和全屏显示比例 (fullScreenRatio)，用于为了在不同物理形态的屏幕上正确渲染视频播放器界面，避免UI元素被手机听筒、前置摄像头等硬件遮挡。

在“用户在视频播放界面点击返回按钮时，应用根据当前屏幕方向状态执行页面回退操作”场景中，我们会从用户交互事件、应用内部状态收集UI 状态变量 (mIsShowCtl, isPortrait)和页面导航栈状态，用于响应用户在视频播放页的返回操作，控制播放器控制栏显示及页面导航逻辑。

在“用户进入搜索页面时，应用从服务器获取热搜榜单数据并展示；当用户点击具体热搜词条或“完整榜单”入口时，将相关关键词或列表数据传递至下一个业务页面”场景中，我们会从网络请求 (ApiManager.requestRanking)、页面路由参数收集热搜榜单数据、默认搜索关键词和用户选中的搜索词，用于实现基于热搜榜的个性化内容推荐与快速跳转，响应用户的浏览和选择意图。

在“用户在应用内执行搜索操作并查看结果时”场景中，我们会从用户输入的搜索关键词、页面跳转携带的路由参数收集搜索关键词 (keyword)和页面路由参数 (router params)，用于响应用户的搜索请求，从服务器检索并返回相关搜索结果。

在“应用启动时初始化设备标识与调试配置”场景中，我们会从本地存储、系统生成收集设备唯一标识符 (appUuid)、调试日志开关 (appLogDebug)和应用包信息 (BundleInfo)，用于设备识别、应用初始化、权限校验及后续内容推荐服务的用户身份关联。

在“热搜页面加载与展示：当用户进入热搜页面时，应用从上一页接收并临时存储热搜榜单数据，用于在界面上渲染热搜关键词及排名列表”场景中，我们会从上游页面传递的路由参数收集热搜榜单数据 (rankings)、搜索关键词 (keyword)和排名位置 (position)，用于获取并展示当前热门搜索内容，响应用户浏览和选择热搜词条的交互需求。

2 设备权限调用

在“应用启动初始化时”场景中，我们会调用ohos.permission.INTERNET权限，用于创建网络连接并注册监听器，以检测设备的互联网连接状态。若您拒绝授权，若拒绝该权限，应用可能无法正确初始化网络连接监听，导致网络状态判断失效或相关功能不可用。

在“将选定的日志文件内容上传至远程服务器”场景中，我们会调用ohos.permission.INTERNET权限，用于建立网络连接以执行 Gist API 的 POST 请求。若您拒绝授权，若拒绝此权限，日志文件将无法上传至云端，导致问题反馈流程中断。此外，在“读取本地存储中的日志文件内容以展示列表及上传”场景中，我们会调用ohos.permission.READ_MEDIA权限，用于访问设备上的日志文件以生成列表供用户选择。若您拒绝授权，若拒绝此权限，应用可能无法扫描或读取本地日志文件，导致列表为空或功能不可用。

在“用户点击上传或自动触发日志同步时”场景中，我们会调用ohos.permission.INTERNET权限，用于将日志内容和元数据上传至 Gitee 云端仓库。若您拒绝授权，无法将日志上传至服务器，反馈提交失败。

在“应用启动时检查并申请访问媒体文件位置信息的权限”场景中，我们会调用ohos.permission.MEDIA_LOCATION权限，用于获取用户媒体文件的位置信息，可能用于基于位置的媒体推荐或管理。若您拒绝授权，若拒绝授权，应用可能无法获取媒体文件的地理位置信息，影响相关功能的正常使用。此外，在“应用启动时检查并申请读取用户媒体文件的权限”场景中，我们会调用ohos.permission.READ_MEDIA权限，用于读取用户相册或文件系统中的媒体文件内容，用于展示或处理。若您拒绝授权，若拒绝授权，应用将无法读取用户的媒体文件，导致无法查看或处理相关内容。

当前已在应用源码/配置扫描或 SDK API 权限映射中识别到以下权限：ohos.permission.GET_NETWORK_INFO；但尚未定位到可回溯的功能点数据流，因此本章节暂不生成具体权限声明。
