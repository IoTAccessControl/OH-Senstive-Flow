1 我们如何收集和使用您的个人信息

在“页面展示时初始化计步功能及获取用户位置信息”场景中，我们会从设备传感器、地理位置服务、本地偏好设置存储收集用户授权状态、位置信息（地点名称）、计步器传感器数据、步数目标与当前步数、错误堆栈与异常详情和后台任务运行状态，用于实现计步器核心功能，记录用户运动步数，显示当前位置信息，并在发生异常时进行调试记录。

在“用户在首页点击开始按钮启动计步功能时，应用进入后台持续运行以收集位置数据并计算步数”场景中，我们会从设备传感器 (加速度计/陀螺仪)、GPS 定位模块收集设备位置信息 (Location)、步数数据 (Steps)和应用启动上下文 (Context)，用于实现计步器核心功能，记录用户运动轨迹和步数统计。

2 设备权限调用

在“页面构建入口”场景中，我们会调用加速度计权限（预授权），用于使用相关系统能力（由 SDK API 权限映射确定）。若您拒绝授权，拒绝授权可能导致对应功能无法正常使用。此外，在“页面构建入口”场景中，我们会调用粗略位置信息权限（动态授权），用于使用相关系统能力（由 SDK API 权限映射确定）。若您拒绝授权，拒绝授权可能导致对应功能无法正常使用。此外，在“页面构建入口”场景中，我们会调用保持后台运行权限（预授权），用于使用相关系统能力（由 SDK API 权限映射确定）。若您拒绝授权，拒绝授权可能导致对应功能无法正常使用。此外，在“页面展示时获取用户当前位置”场景中，我们会调用位置信息权限（动态授权），用于获取设备的地理位置信息及地址名称，用于展示当前所在位置。若您拒绝授权，无法获取位置信息，页面将无法显示当前地点，相关定位功能失效。

在“开始/停止按钮”场景中，我们会调用保持后台运行权限（预授权），用于使用相关系统能力（由 SDK API 权限映射确定）。若您拒绝授权，拒绝授权可能导致对应功能无法正常使用。此外，在“用户在首页点击开始按钮启动计步功能时”场景中，我们会调用位置信息权限（动态授权），用于获取设备实时地理位置信息，用于后台持续追踪用户移动轨迹以计算步数。若您拒绝授权，无法获取位置信息，导致计步功能无法正常工作，无法记录用户的运动数据和轨迹。

当前已在应用源码/配置扫描或 SDK API 权限映射中识别到以下权限：运动状态识别权限（动态授权）、后台位置信息权限（预授权）；但尚未定位到可回溯的功能点数据流，因此本章节暂不生成具体权限声明。
