1 我们如何收集和使用您的个人信息

在“分类详情页面加载时显示加载提示弹窗”场景中，我们会从应用内部组件状态收集UI 状态日志 (LoadingTips)和组件生命周期事件，用于记录加载弹窗的展示状态以辅助开发调试。

在“分类详情页面加载展示时，自动获取系统窗口布局信息以适配界面显示”场景中，我们会从系统窗口服务收集系统状态栏高度、底部导航栏高度和窗口避让区域信息，用于获取系统状态栏和导航栏尺寸，用于调整页面顶栏及列表的布局位置，确保内容不被系统UI遮挡。

在“用户在分类详情页面查看视频列表并触发视频组件加载与播放时”场景中，我们会从应用内部状态数据、系统日志接口收集UI 组件控制状态 (showControls)、UI 组件加载状态 (showLoading)、视频播放地址 (playUrl)、自动播放配置 (isAutoPlay)和日志格式化参数 (domain, prefix, format, args)，用于实现视频组件的界面展示、播放控制逻辑以及记录应用运行时的调试信息。

在“分类详情页面加载时初始化骨架屏及执行路由跳转”场景中，我们会从应用内部状态变量、系统路由服务收集骨架屏占位数据列表、页面路由 URL、路由参数对象、路由错误码和路由错误信息，用于渲染页面加载骨架屏、执行页面导航及记录路由跳转状态与异常信息。

在“用户进入分类详情页面时，应用接收并处理页面跳转参数以展示对应的分类标题和头部图片”场景中，我们会从页面路由参数收集分类 ID、分类标题和分类头部图片地址，用于初始化页面内容，展示当前分类的详细信息列表。此外，在“用户在分类详情页进行交互或触发重试逻辑时，应用执行页面跳转操作”场景中，我们会从内部业务逻辑、用户交互事件收集目标页面 URL和路由跳转参数，用于响应用户操作或网络重试需求，实现页面间的导航跳转。此外，在“应用在页面跳转过程中发生错误或成功完成跳转时，记录诊断日志”场景中，我们会从系统错误回调、路由执行结果收集路由错误码、路由错误信息和跳转成功的 URL，用于记录应用运行时的异常情况和导航状态，用于故障排查和调试。此外，在“应用从网络加载分类详情列表数据并更新到页面视图”场景中，我们会从网络请求响应收集分类列表数据和视图状态数据，用于展示分类下的具体内容列表，支持用户浏览和交互。

在“用户进入分类详情页面时，应用自动加载并展示该分类的标题、头图及列表内容”场景中，我们会从路由参数、网络请求收集分类 ID、分类头图 URL、分类标题和分类详情列表数据，用于根据用户选择的分类，动态获取并展示对应的商品或内容列表及相关元数据。

在“用户在分类详情页面下拉刷新或点击重试按钮时，应用重新加载并展示最新的业务数据列表”场景中，我们会从用户交互事件（下拉刷新、重试）、网络请求返回的数据收集视图状态 (viewState)、刷新状态标志 (isRefreshing)、业务数据列表 (dataList)、滚动偏移量 (scrollOffset)、路由跳转参数 (params)、目标页面 URL和错误码与错误信息，用于响应用户操作以更新页面内容，展示最新的分类详情数据，并记录必要的调试与错误日志以便排查问题。

在“用户在分类详情页面点击列表项时，应用根据用户选择的目标内容执行页面跳转操作”场景中，我们会从用户交互事件、本地内存数据收集目标页面 URL、页面路由参数 (params)和业务列表数据 (HomeModelIssueListItemList)，用于响应用户点击操作，实现从当前分类详情页面向目标详情页的页面切换与数据传递。

在“用户在分类页面浏览内容或点击分类项进行页面跳转时，系统记录路由跳转状态（成功或失败）及相关的调试信息”场景中，我们会从用户交互行为（点击、导航）、应用内部路由逻辑收集目标路由 URL、路由跳转参数、路由跳转错误码、路由跳转错误信息和操作日志字符串，用于开发调试、故障排查及监控路由跳转的异常情况。

在“用户在分类页面浏览分类列表，点击分类项进入详情页面，或触发数据刷新重试时”场景中，我们会从网络请求 (CategoryViewModel.getCategoryDataList)、用户交互事件 (点击/重试)收集分类数据列表 (CategoryModelChild)、背景图片资源路径 (bgPicture)、分类名称 (name)、页面路由参数 (params)、目标页面 URL、路由错误信息 (err.code, err.message)和路由成功日志信息，用于展示分类列表内容，响应用户点击进行页面导航，以及记录应用运行过程中的路由跳转状态和异常信息以便调试。

在“用户进入分类页面时，应用加载并展示分类列表及其背景图片”场景中，我们会从网络请求返回数据、页面路由参数收集分类数据列表 (CategoryModelChild)、背景图片 URL (bgPicture)和页面路由参数 (params)，用于在分类页面展示分类项及其对应的背景图片和名称。

在“发现容器页加载时，获取页面路由参数以确定展示内容类型”场景中，我们会从应用内部路由参数收集页面路由参数 (containerType)，用于根据传入的类型参数动态渲染页面标题及内容。

在“热门页面加载与导航交互”场景中，我们会从应用内部状态变量、用户交互事件收集热点标签列表数据、页面路由参数、目标页面 URL、UI 组件状态数据 (当前选中索引、标签列表)、用户 Tab 切换位置索引和路由错误码与错误信息，用于初始化热门页面内容、响应用户导航操作、记录页面跳转状态及调试信息以辅助开发维护。

在“用户进入视频详情页时，应用从页面跳转参数中获取视频标识和播放地址以加载内容”场景中，我们会从页面路由参数收集视频关联标识 (relationId)和视频播放地址 (playUrl)，用于在视频详情页展示对应的视频内容及标题信息。

在“用户在发现页面查看焦点图组件时，应用加载并展示列表项的图标、标题及封面图片；当用户点击列表项触发页面跳转时，应用记录路由跳转的目标地址及操作状态日志”场景中，我们会从内存中的业务数据模型 (FocusModelItemList)、系统路由模块收集页面路由 URL、路由参数对象 (params)、业务数据项 (FocusModelItemList)、图片资源路径 (icon, cover.feed)、文本标题 (title)和路由跳转错误码与消息，用于支持页面间的导航跳转功能，并在发生异常或完成操作时记录调试日志以便追踪问题。

在“首页加载及页面跳转时，记录路由导航状态、目标地址及错误信息用于调试与监控”场景中，我们会从应用内部路由逻辑、系统路由服务收集页面路由 URL、路由参数对象 (params)、路由错误码、路由错误信息和操作日志，用于记录页面跳转轨迹、诊断路由失败原因及监控应用运行状态。

在“用户在首页下拉刷新或点击视频列表项时，应用会收集并传递视频内容 ID 和播放地址以进行页面跳转”场景中，我们会从用户操作（下拉刷新、点击列表项）收集视频内容 ID、视频播放地址和路由跳转错误信息，用于实现从首页列表页到视频详情页的页面导航与数据回传。

在“用户在首页动态流列表中点击视频条目时，应用提取该条目的 ID 和播放地址以跳转至详情页；若跳转失败或成功，系统会记录相关日志”场景中，我们会从用户交互（点击列表项）、应用内部数据模型收集视频内容 ID、视频播放地址、路由跳转错误码、路由跳转错误信息和页面路由路径，用于实现应用内页面导航，将用户选中的视频详情数据传递给目标页面进行展示；记录路由操作状态以便调试和故障排查。

在“用户在首页点击视频条目时，应用提取该视频的 ID 和播放地址，用于跳转至详情页进行内容展示”场景中，我们会从用户交互（点击列表项）、本地内存数据（this.dataList）收集视频唯一标识 (id)、视频播放地址 (playUrl)、页面路由路径和路由操作日志信息，用于实现应用内页面导航，将用户选中的视频详情数据传递至目标页面以进行展示，并记录路由跳转状态以便调试。

在“热门榜单页面展示与隐藏时记录系统日志”场景中，我们会从应用内部事件收集页面生命周期状态事件和静态日志字符串常量，用于开发调试与运行状态监控。

在“主页面加载与路由跳转时的状态提示管理”场景中，我们会从应用内部事件总线、系统路由模块收集UI 状态标志 (loadingShow)、加载提示文本 (loadingTips)、事件负载数据 (event.data)、页面路由 URL、路由参数对象 (params)、路由跳转错误信息和系统日志信息，用于在页面加载或跳转过程中向用户展示加载状态提示，并在发生路由错误时记录调试信息以辅助问题排查。

在“用户进入或离开“我的页面”时，应用记录页面状态变化日志”场景中，我们会从系统生命周期事件收集页面生命周期状态事件和静态日志字符串常量，用于开发调试与运行状态监控。

在“用户进入‘我的’页面时，系统加载页面布局、默认头像及应用名称文本。若发生页面间跳转或路由操作，会记录相关状态日志”场景中，我们会从应用内部路由参数、UI 渲染事件收集页面路由 URL、路由参数对象 (params)、路由跳转错误码与消息和UI 组件构建事件，用于诊断应用内的页面跳转逻辑是否正常，以及记录 UI 渲染过程中的关键节点状态，便于问题排查。

在“用户查看排行榜列表并点击条目跳转至视频详情页时”场景中，我们会从应用内部数据（排行榜列表）、用户交互行为（点击事件）收集页面路由 URL、路由参数对象 (params)、视频 ID、视频播放地址、错误码与错误信息和操作日志，用于实现页面间导航跳转以展示视频详情，以及记录路由跳转过程中的成功或失败状态用于调试与监控。

在“页面加载与初始化时，应用会收集内部滚动列表的长度统计信息（如 scrollList.length），用于调试和状态监控”场景中，我们会从本地内存数据收集scrollList 长度 (UI 状态数据)，用于开发调试、监控 UI 组件的初始化状态及列表渲染情况，不向第三方传输。

在“页面加载时初始化测试数据列表，以及用户进行横向滚动交互时记录调试日志”场景中，我们会从应用内部生成的模拟数据、用户界面交互事件（滚动）收集本地列表索引数据、UI 滚动偏移量、路由跳转参数与 URL和系统错误码与错误信息，用于开发调试，记录页面初始化状态、路由跳转情况及用户滚动交互行为以便排查问题。

在“用户点击测试页面中的模拟登录按钮时，应用执行本地登录状态与用户信息的保存操作”场景中，我们会从用户交互输入（点击事件）、硬编码的测试数据收集用户登录状态 (USER_LOGIN)和用户名/用户标识 (USER_INFO_KEY)，用于在测试环境中模拟用户登录行为，以便验证后续功能逻辑。此外，在“测试页面加载初始化时，应用自动从云端获取首页 Banner 数据并输出调试日志”场景中，我们会从网络请求响应数据收集网络响应数据 (WanAndroid Banner)，用于开发调试阶段用于验证网络接口连通性及数据格式正确性。

在“页面展示时自动加载首页 Banner 数据”场景中，我们会从网络响应收集网络响应数据 (Home Banner)和JSON 格式化的业务数据，用于调试与开发测试，用于验证网络请求及数据返回情况。

在“用户进入话题详情页时，应用加载并展示该话题的标题、简介及列表内容”场景中，我们会从页面路由参数、云端服务器收集话题标识 (topicId)、话题详情数据 (detailData)和话题标题/简介 (title/brief)，用于根据用户选择的话题 ID，从服务器获取并展示对应的话题详情信息（包括标题、简介及列表）。

在“用户在话题页面浏览或点击内容触发页面跳转时，系统记录路由跳转的成功状态、目标地址及失败时的错误信息”场景中，我们会从应用内部路由逻辑、用户交互事件收集路由跳转目标 URL、路由跳转错误码、路由跳转错误信息和路由参数，用于记录应用内部页面导航的调试信息与异常排查，辅助开发人员定位路由跳转问题。

在“用户在话题页面点击重试按钮或下拉刷新时，应用重新加载话题列表数据；或在路由跳转过程中记录操作日志”场景中，我们会从用户交互（点击重试、下拉刷新）、网络请求返回的数据、系统路由参数收集话题列表数据、图片资源路径、页面路由参数、目标页面 URL、错误日志信息和UI 状态数据，用于支持话题内容的展示、刷新及重试功能，并记录路由跳转与错误信息以辅助调试和监控。

在“用户在话题页面下拉刷新或点击列表项时，应用加载并展示主题数据，执行页面跳转，并记录操作日志”场景中，我们会从用户交互事件（下拉刷新、点击）、网络请求返回的业务数据、系统路由模块收集主题列表数据 (topicList)、图片资源地址、页面路由参数 (params)、目标页面 URL、UI 状态数据 (isRefreshing)、错误日志信息 (err.code, err.message)和成功日志信息 (URL)，用于实现话题列表的展示与刷新，响应用户点击进行页面跳转，以及记录应用运行状态和错误信息用于调试。

在“用户在话题列表页面浏览并点击特定话题项时，应用执行页面跳转操作”场景中，我们会从用户交互事件、内存中的业务数据收集页面路由 URL、路由参数对象 (params)、错误码与错误信息、主题列表数据 (topicList)和图片资源路径，用于实现话题列表的渲染展示以及用户点击后的页面导航跳转功能。

2 设备权限调用

在“用户在分类详情页面进行下拉刷新或点击重试时，应用需要访问网络以获取最新的业务数据”场景中，我们会调用网络访问权限权限，用于允许应用发起网络请求，从服务器拉取分类详情列表数据。若您拒绝授权，若拒绝此权限，用户将无法通过下拉刷新或重试功能获取最新数据，页面可能显示为空或保留旧数据。

在“测试页面加载初始化时，应用需要从远程服务器获取首页 Banner 数据”场景中，我们会调用网络访问权限权限，用于允许应用发起 HTTP 网络请求，从云端下载业务数据。若您拒绝授权，若拒绝或无此权限，应用无法获取 Banner 数据，导致页面内容缺失且调试日志无法记录相关数据。

在“页面展示时自动加载首页 Banner 数据”场景中，我们会调用网络访问权限权限，用于允许应用发起网络请求以获取远程服务器上的 Banner 数据。若您拒绝授权，无法加载首页 Banner 内容，相关功能可能显示为空或失败。
